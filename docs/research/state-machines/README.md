# State Machine — Reference Matrices

Three matrices that document the integration's state model:

- [Doc 1 — Per-dimension transition matrix](transitions.md) — every input event that can flip each state-machine dimension.
- [Doc 2 — Reboot survival + idle value matrix](reboot-and-idle.md) — what each field looks like at cold-start and after restart; prescriptive idle values.
- [Doc 3 — Entity → field dependency matrix](entity-sources.md) — generated by `tools/state_machine_audit.py`.

The audit verifier at `tools/state_machine_audit.py` runs three checks per entity (sourcing / idle / reboot) against these matrices. When the verifier shows all green the integration's state model is clean.
